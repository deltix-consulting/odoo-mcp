# odoo_mcp_long_lived_keys

Small Odoo addon. Adds a per-user **"Allow long-lived MCP API keys"** checkbox to the user form. When ticked, API keys generated by that user get a 90-day expiry instead of the 1-day default that some Odoo installs enforce for non-admin users.

## Why this exists

Some Odoo installs (Odoo Online, deployments with `auth_password_policy`, certain custom modules) force non-admin API keys to expire after 1 day. That's a reasonable default policy for password-like credentials, but it's incompatible with MCP-style integrations where the user generates one key and uses it for weeks.

This addon gives an Odoo admin a one-click way to opt specific users out of that default without granting them admin rights or weakening the global policy.

## Install

1. Copy `odoo_mcp_long_lived_keys/` into your Odoo addons path (alongside the other modules you've installed).
2. Restart Odoo with `--update=all` once, or use the UI: **Apps → Update Apps List → search "MCP Long-Lived API Keys" → Install**.
3. Open any user form: **Settings → Users & Companies → Users → click a user → Access Rights tab**. You should now see an **MCP Integration** section with the **"Allow long-lived MCP API keys"** checkbox.

## Use

For each colleague who runs the odoo-mcp integration:

1. Open their user record.
2. Tick **"Allow long-lived MCP API keys"**.
3. Save.
4. Ask the colleague to **generate a new API key** (their existing 1-day key is not retroactively changed — they need to delete the old one and create a new one).

The new key will have a 90-day expiry. Verify by checking the "Expires on" column in their **Profile → Account Security** tab.

## Revoke

Untick the checkbox. Their **next** key reverts to the 1-day default. Existing long-lived keys keep their 90-day expiry — to revoke immediately, delete the specific key in **Account Security**.

## Customise

Edit `_LONG_LIVED_DURATION_DAYS` at the top of `models/res_users.py`. Reasonable values: 30, 60, 90, 180, 365. Avoid "Never" — manual rotation is part of the security story.

## Status

**Tested locally only by the maintainers.** Install on a non-production Odoo first to confirm:

- The checkbox appears on the user form
- Ticking it then generating a new API key produces a 90-day expiry
- Unticking it then generating a new API key produces the previous (1-day) default
- The override doesn't break anything for admin users or for users who already had their own expiration policy
